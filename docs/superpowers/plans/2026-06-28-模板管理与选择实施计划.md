# 模板管理与选择 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为桌面端补齐全局模板、项目模板、模板管理弹窗、项目级模板选择记忆，并打包生成 Windows exe。

**Architecture:** 模板作为正式桌面资源通过 IPC 读写，由主进程负责持久化模板和项目选择偏好，渲染进程只负责上传文件、编辑表单和选择状态。提取任务仍传 `templateIds`，主进程在执行任务时按当前书籍所属项目解析模板正文并拼入 prompt。

**Tech Stack:** Electron + React + TypeScript + Vitest + Playwright + electron-builder。

## Self Review

- 通用架构：模板是独立资源，不绑定到单个表单组件。
- 设计模式适配：主进程负责持久化和 prompt 解析，渲染进程负责 UI 状态和文件读取。
- 配置化覆盖范围：内置模板继续来自配置，用户模板进入模板资源存储，项目选择进入项目偏好存储。
- Scope：本计划不实现复杂模板市场、版本历史或模板正文在线富文本编辑，只实现用户要求的上传、编辑、删除、全局/项目区分和选择记忆。
